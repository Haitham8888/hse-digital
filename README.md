# HSE Digital — نظام إدارة السلامة الرقمي

منصة ويب تحوّل أوراق السلامة في المشاريع الإنشائية (تصاريح العمل، تشيك لست المعدات، تقييم المخاطر)
إلى نماذج رقمية بموافقات وتواقيع إلكترونية متسلسلة وأرشفة تلقائية في Google Drive — صفر ورق.

مبنية على نماذج مشروع SCC (مصنع القهوة السعودية — جازان) بين المقاول RBC والاستشاري ESCC.

**الموقع:** https://haitham8888.github.io/hse-digital/

## الوحدات

| الوحدة | ما تقدمه |
|---|---|
| **تصاريح العمل (PTW)** | إنشاء من الجوال، كود مركزي تلقائي `SCC-RBC-ESCC-{مبنى}-PTW.{G/H}-{تسلسل}` بلا تعارض بين المستخدمين، تشيك لست YES/NO/N/A، تمديد وإغلاق إلكتروني |
| **محرك الموافقات** | سلسلة: مدير الموقع ← مشرف HSE ← مهندس الاستشاري ← استشاري HSE، توقيع بالإصبع أو محفوظ مختوم بالاسم والوقت، رفض مع ملاحظة، حماية كل دور برمز PIN |
| **الإشعارات** | إيميل تلقائي للمعتمد الذي عليه الدور برابط مباشر للتصريح، وإيميل للمنشئ عند الاعتماد النهائي أو الرفض + زر تنبيه واتساب على رقم صاحب الدور |
| **الأرشفة** | PDF رسمي مطابق للنموذج المعتمد، يُحفظ تلقائيًا في مجلد `HSE Archive (PDF)` داخل مجلد المشروع في Drive عند إغلاق التصريح (وزر أرشفة يدوي) |
| **فحص المعدات + QR** | نماذج فحص لكل نوع معدة، QR يفتح نموذج الفحص، إيقاف تلقائي عند أي بند NO، كود اللون الشهري وملصق قابل للطباعة |
| **تقييم المخاطر** | اقتراح المخاطر والاحتياطات تلقائيًا من المكتبة حسب النشاط، تقييم شدة×احتمالية 5×5 |
| **إدارة الموظفين** | إضافة/تعديل/إيقاف بالأدوار والجهات، جوال (واتساب) وبريد (إشعارات)، سلسلة الاعتماد تقرأ من السجل |
| **المزامنة** | قاعدة بيانات مشتركة (Google Sheet) لكل الأجهزة، تعمل أوفلاين ويُرفع كل شيء تلقائيًا عند عودة الاتصال، سحب دوري للتحديثات |

## البنية

```
index.html        هيكل التطبيق
css/app.css       نظام التصميم + قالب طباعة A4
js/config.js      رابط الخلفية (backendUrl)
js/data.js        القوالب المرجعية والبيانات التجريبية
js/cloud.js       محرك المزامنة (طابور أوفلاين، أكواد مركزية، أرشفة، PIN)
js/app.js         المنطق والشاشات
backend/Code.gs   الخلفية: Google Apps Script (قاعدة Sheet + أرشيف Drive + إيميلات)
```

**القاعدة:** Google Sheet باسم `HSE Digital — Database` داخل مجلد المشروع في Drive —
تبويب لكل كيان (Permits يعمل كسجل PTW LOG رسمي قابل للفرز والفلترة) + Counters + Settings.

## تفعيل الخلفية (مرة واحدة — ٥ دقائق)

1. افتح **script.google.com** ← مشروع جديد ← الصق محتوى `backend/Code.gs`
2. عبّئ `FOLDER_ID` بمعرّف مجلد ملفات المشروع في Drive (ما بعد `/folders/` في الرابط)
3. شغّل الدالة **setup** مرة واحدة ووافق على الأذونات — ستُنشأ القاعدة والأرشيف تلقائيًا
4. **Deploy ← New deployment ← Web app** واضبط: `Execute as: Me` و `Who has access: Anyone` ثم انسخ الرابط
5. ضع الرابط في `js/config.js` ← `backendUrl` وارفع التعديل — ينشر تلقائيًا

بدون هذه الخطوة يعمل الموقع بوضع تجريبي محلي (بيانات كل جهاز على حدة).

## قائمة ما قبل الإطلاق

- [ ] غيّر رموز PIN الافتراضية من تبويب **Settings** في الـ Sheet (الافتراضية: 1111/2222/3333/4444)
- [ ] أدخل الموظفين الفعليين (بريد + جوال) من شاشة إدارة الموظفين
- [ ] اضبط عدادات الأكواد في تبويب **Counters** لتكمل ترقيمكم الورقي الحالي
- [ ] جرّب دورة كاملة: إنشاء ← 4 تواقيع ← إغلاق ← تحقق من وصول الـ PDF في مجلد الأرشيف
