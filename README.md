# HSE Digital — نظام إدارة السلامة الرقمي (MVP)

منصة ويب تحوّل أوراق السلامة في المشاريع الإنشائية (تصاريح العمل، تشيك لست المعدات، تقييم المخاطر)
إلى نماذج رقمية بموافقات وتواقيع إلكترونية متسلسلة وأرشفة تلقائية — صفر ورق، صفر لف على المكاتب.

مبنية على نماذج مشروع SCC (مصنع القهوة السعودية — جازان) بين المقاول RBC والاستشاري ESCC.

## الوحدات

| الوحدة | ما تقدمه |
|---|---|
| **تصاريح العمل (PTW)** | إنشاء من الجوال، توليد الكود تلقائيًا `SCC-RBC-ESCC-{مبنى}-PTW.{G/H}-{تسلسل}`، تشيك لست YES/NO/N/A (10 بنود للعام، 12 للأعمال الساخنة)، تمديد وإغلاق إلكتروني |
| **محرك الموافقات** | سلسلة معتمدين: مدير الموقع ← مشرف HSE ← مهندس الاستشاري ← استشاري HSE، توقيع بالإصبع أو توقيع محفوظ مختوم بالاسم والوقت، رفض مع ملاحظة، تنبيه واتساب للمعتمد الذي عليه الدور |
| **PDF رسمي** | طباعة/تصدير A4 مطابق للنموذج المعتمد (الأطراف الثلاثة، التشيك لست، التواقيع، التمديد والإغلاق) مع ختم الحالة |
| **فحص المعدات + QR** | نماذج فحص لكل نوع معدة، QR على كل معدة يفتح نموذج الفحص مباشرة، سجل فحوصات، إيقاف تلقائي عند أي بند NO، كود اللون الشهري وطباعة الملصق |
| **تقييم المخاطر** | تكتب النشاط والنظام يقترح المخاطر والاحتياطات من المكتبة، تقييم شدة×احتمالية 5×5 |
| **إدارة الموظفين** | إضافة/تعديل/إيقاف الموظفين بالأدوار (معتمدون، فاحصون، عمال)، اختيار العمال في التصريح من السجل، رقم جوال لكل معتمد ليصله تنبيه واتساب مباشرة، سلسلة الاعتماد تقرأ الأسماء من السجل |
| **لوحة التحكم** | التصاريح السارية، قيد الموافقة، «بانتظار توقيعك» حسب دورك، المعدات المتأخرة عن الفحص |

## التشغيل

الواجهة ثابتة بالكامل (HTML/CSS/JS بدون بناء):

```bash
python3 -m http.server 8000
# ثم افتح http://localhost:8000/
```

أو فعّل GitHub Pages (Settings ← Pages ← Branch: main / root) ليكون الموقع متاحًا من أي جوال.

- البيانات التجريبية تُبذر تلقائيًا عند أول تشغيل وتُحفظ في LocalStorage على الجهاز.
- مبدّل الدور أعلى الصفحة يحاكي دخول كل شخص في سلسلة الموافقات (منشئ / مدير موقع / مشرف HSE / مهندس استشاري / استشاري HSE).
- زر «إعادة تعيين البيانات التجريبية» أسفل الصفحة يعيد كل شيء لحالته الأولى.

## البنية

```
index.html    هيكل التطبيق + شريط التنقل
css/app.css   نظام التصميم (ورق وحبر + أصفر السلامة) وقالب طباعة A4
js/data.js    القوالب المرجعية (التشيك لست، سلسلة الموافقات، مكتبة المخاطر) والبيانات التجريبية
js/app.js     المنطق: التوجيه، الموافقات، لوحة التوقيع، إدارة الموظفين، الطباعة، QR
```

## خارطة الطريق (بعد الـ MVP)

1. خلفية حقيقية (Node.js + PostgreSQL) بدل LocalStorage، وحسابات مستخدمين بصلاحيات.
2. إشعارات WhatsApp Business API / بريد فعلية برابط موافقة مباشر.
3. رفع نسخة PDF تلقائيًا إلى Google Drive للأرشفة.
4. تقارير وإحصائيات (زمن دورة الموافقة، الأكثر رفضًا، التزام الفحص الشهري).
